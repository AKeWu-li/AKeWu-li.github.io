## 开始

### 定义

计算机软件通常分为两类，即系统软件和应用软件。系统软件用于计算机的管理、维护、控制和运行以及对运行的程序进行相关的翻译、装入等服务工作。系统软件通常分为三部分，即操作系统、语言处理系统和常用的例行服务程序。

```language
操作系统是计算机系统的一种系统软件，它用于管理计算机的资源和控制程序的执行。一个程序只有在通过操作系统获得必需的资源后才能执行。
```

### 基本功能

#### 人机交互界面

通常操作系统需要提供人机交互界面功能为用户提供各种计算服务

```language
用户可直接使用键盘命令或Shell命令语言或图形用户界面(GUI)与计算机交互，表达计算需求，由操作系统人机交互界面功能模块分析处理并调用系统内部模块，最终完成用户的计算任务并将结果反馈。
```

#### 资源管理

从资源管理的观点来看，操作系统的主要功能主要包括：处理器与进程管理、存储器管理、文件管理、设备管理和网络管理

### 分类

#### 单用户操作系统

```language
一个用户独占计算机资源，系统所有软硬件资源全为一个用户服务，单独执行该用户提交的一个任务。
```

#### 批处理操作系统

又分为单道批处理和多道批处理系统，区别在于

```language
(1)作业道数。单道批处理系统只有一道作业在主存中运行，而多道批处理中同时有多道作业在运行。
(2)作业处理方式。单道批处理是将多个用户作业形成一批，由卫星机将这些作业输入磁带中，然后主机再从该磁带中将作业一个一个地读入主存进行处理。作业完成后，将结果也输入到另一磁带中去，当这批作业全部完成后，再由卫星机把此磁带上的结果通过相应的输出设备输出。处理完一批设备再处理另一批作业。多道批处理系统中作业可随时被接收进入系统，并存放在磁盘输入池充中形成作业队列，而后操作系统按一定原则从作业队列中调入一个或多个作业进入主存运行。
```



批操作：用户与他的作业之间没有交互作用，不能直接控制其作业的运行。

#### 实时操作系统

```language
实时系统是较少有人为干预监督和控制的系统。它的一个基本特征是事件驱动设计，即当接收了某种类型的外部消息后，由系统选择一个程序去执行。
```

#### 分时操作系统

```language
指多个用户分享使用同一台计算机，把计算机的系统资源进行时间上的分割，将CPU整个工作时间分成一个个的时间段，每个时间段称为一个时间片。让每个用户轮流使用这些时间片，使得每个用户均感到自己独享该CPU。
```

```language
主要目的是对联机用户的服务和响应，主要特点为：同时性、独立性、及时性、交互性。
```

```language
分时操作系统与批处理操作系统的差别
(1)批处理操作系统的目标是提高机器效率；而分时操作系统的目标是对用户请求的快速响应。
(2)批处理多道操作适合需要较长时间才能完成的大型作业；而分时系统适用于要求在几分钟内能从终端上获得结果的短小作业。
(3)批处理多道操作对于充分利用系统资源而言较好；而分时操作系统对于要求执行相同功能的作业而言较好。
```



#### 网络操作系统

```language
提供网络通信和网络资源共享功能的操作系统称为网络操作系统。
具有如下额外功能：（除处理器与进程管理、存储器管理、设备管理和文件管理之外）
(1)提供高效、可靠的网络通信能力。
(2)提供多种网络服务功能。
```



#### 分布式操作系统

```language
一种特殊的网络操作系统，是由多台计算机组成的网络并满足以下条件的系统：
(1)系统中任意两台计算机可以通过通信来交换信息。
(2)系统中各台计算机无主次之分，既没有控制整个系统的主机，也没有受控于他机的主机。
(3)系统的资源为所有用户共享。
(4)系统中若干台计算机可以相互协作来完成一个共同任务，或者说，一个程序可以分布于几台计算机上并行地运行。
```



#### 嵌入式操作系统

```language
一种支持嵌入式应用的操作系统，通常包括以下几个模块：与硬件相关的底层驱动软件、系统内核、设备驱动接口、通信协议、图形界面、标准化浏览器等。
```

特点

```language
(1)能够有效管理复杂的系统资源；
(2)能够对硬件进行抽象；
(3)能够提供库函数、驱动程序、开发工具集等。
```



```language
与通用操作系统相比较，嵌入式操作系统具有的特性如下：
(1)系统具有实时性；
(2)系统具有硬件依赖性；
(3)系统软件具有固化性；
(4)系统具有应用专用性；
(5)系统具有可裁剪性；
```

### 现代操作系统

#### 特点

```language
(1)从使用方式上来说，绝大多数是个人独占计算机，通常使用交互方式，并能联网运行。
(2)由于对系统可靠性、灵活性或可扩充性的要求，多机系统，尤其对称多处理器SMP方式将被广泛应用。
(3)在操作系统的设计技术和结构上的改进和发展主要体现在这些方面：1.微内核结构；2.多线程机制；3.对称多处理器机制；4.分布式操作系统；5.操作系统设计时普遍采用面向对象技术以及软件工程的其他原则。
```
