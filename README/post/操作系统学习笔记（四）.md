## 开始

### 处理器调度层次

```language
一般分为三个级别：长程调度、中程调度和短程调度。长程调度的对象是作业，短程调度的对象是进程（或内核级线程）。在纯粹的分时或实时操作系统中，通常不需要配备长程调度；在分时系统或具有虚拟存储器的操作系统中，为了提高内存利用率和作业吞吐量，专门引进了中程调度。
```

#### 长程调度（高级调度、宏调度、作业调度）

```language
发生在新进程的创建中，它决定一个进程能否被创建，或者是创建后能否被置成就绪状态，以参与竞争处理器资源获得运行。
```

```language
在批处理操作系统中，作业首先进入系统在辅存上的后备作业队列等候调度，因此长程调度是必须的，它的执行频率较低，并与到达系统的作业的数量和速率有关。
```

##### 主要功能

```language
根据作业控制块中的信息，按照某种原则从外存上的后备队列中选取一个或几个作业调入内存，并为他们创建进程、分配必要的资源，然后再将新创建的进程插入就绪队列。
```



#### 中程调度（中级调度）

```language
反映到进程状态上就是挂起和解除挂起，它根据系统当前的负荷情况决定停留在主存中的进程数。
```

##### 主要功能

~~~language
负责内外存之间的进程对换，以解决内存紧张的问题，即它将内存中处于等待状态的某些进程调到外存对换区，以腾出内存空间，再将外存对换区中已具备运行条件的进程重新调入内存准备运行。
~~~



#### 短程调度（低级调度、进程调度）

```language
各类操作系统必须具有的功能。
```

```language
执行短程调度功能的程序是进程调度程序，程序的执行频率很高，它必须常驻内存。进程调度是操作系统中最基本的调度，在批处理、分时和实时操作系统中都必须配置它。
```

##### 主要功能

```language
决定就绪队列中哪个进程或线程将获得处理器，并实际执行将处理器分配给该进程或者线程的工作。
```

### 调度准则

```language
(1)资源利用率
(2)平衡资源
(3)响应时间
(4)周转时间
(5)吞吐率
(6)公平性
```

### 实时调度

#### 特点

```language
(1)有限等待时间（决定性）
(2)有限响应时间
(3)用户控制
(4)可靠性高
(5)系统处理出错能力强
```

上述特性要求它必须具有如下三方面的处理能力

```language
(1)快速的进程和线程切换速度
(2)快速的外部中断响应能力
(3)基于优先级的随时抢先式调度策略
```



#### 分类

```language
(1)根据实时任务性质的不同，分为硬实时调度算法和软实时调度算法
(2)根据调度方式的不同，分为非抢占调度算法和抢占调度算法
(3)根据调度程序调度时间的不同分为静态调度算法和动态调度算法
```

以下根据（1）系统是否执行可调度性分析；（2）静态还是动态；（3）是否会根据运行时分派的任务产生一个调度或计划，这三个标准分类

```language
(1)静态表格驱动类
(2)静态优先级驱动抢占调度类
(3)基于动态规划调度的调度类
(4)动态尽力调度类
```
